## TASK1
Initialize, Commit, and Branch Basics

## REQUIREMENTS
- Use `git init`, `git add`, and `git commit` to start your repository.
- Create a branch using `git branch` or `git checkout -b` and switch between branches.
- Merge the branch back to the main branch and verify the commit history.

## TECHNOLOGIES USED
git

## STEPS 

- Initialse the Git 

COMMAND - git init

Here a .git folder will be created

Create a git repository

Repository is a folder which stores the commits,history,branches,files.

Default branch is master - If needed we can change

- ADD File (Stagging Area)

COMMAND - git add README.md

Add README file

Moves to stagging area

- COMMIT (ADD To Local Repository)

COMMAND - git commit -m "Commit Message"

Moves to Local Repository

Created with a Commit ID

- RENAMING THE BRANCH

COMMAND - git branch -M main

master default branch is renamed to main

- ADD THE REMOTE REPO (GITHUB)

COMMAND - git remote add origin https://github.com/SivaKavindraTamilselvan/Git.git

connected to the github

- PUSH TO REMOTE REPO

COMMAND - git push -u origin main

Local main is pushed to the remote main (GITHUB)

REMOTE REPO - origin/main

origin - remote name

- CREATE NEW BRANCH

COMMAND - git checkout -b Task1

Created new branch

Moves to the new branch

- ADD FILE

- COMMIT THE FILES

- PUSH TO THE TASK1 BRACH

- CHECKOUT TO MAIN

- MERGE TO MAIN

COMMAND - git merge Task1

Task1 remote repo merged to main repo
